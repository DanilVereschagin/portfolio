Blog

Blog - веб-приложение блога, демонстрирующая функционал Next.js. Ссылка на код - "Перейти".

Главная страница и хедер от лица гостя

На главной странице только картинка с котиком. Также виден хедер от роли гостя. Есть кнопка авторизации, а также отсутствует кнопка перехода на страницу профиля.

Страница авторизации

Пользователь может авторизоваться на сайте с помощью аккаунта Google или через свою почту и пароль.

Страница профиля

На странице профиля отображается информация о имени пользователя, а также аватарка, если пользователь зашёл через Google.

Страница блога

На данной странице пользователь может посмотреть список постов, которые были опубликованы пользователями. Можно открыть конкретный пост. Также присутствует поиск по названию поста и его содержимого.

Демонстрация работы поиска

При нажатии на кнопку "Поиск" отображается список постов, которые содержат в названии или содержимом ключевое слово, которое пользователь ввел в поле поиска.

Страница поста

На данной странице пользователь может посмотреть наименование поста, его содержимое, а также возможность изменить пост или удалить его. Принадлежность поста пользователю не реализовывалась.

Страница создания нового поста

На данной странице пользователь может создать новый пост.

Страница редактирования поста

На данной странице пользователь может редактировать существующий пост.

Используемые технологии

Ниже представлены технологии, используемые для данного проекта.

CSS3

Использовался только кастомные css стили.

TypeScript

Весь код проекта был написан на TypeScript.

EsLint

Весь код проекта был проверен с помощью EsLint.

JSON-server

Для хостинга псевдо-БД использовалась библиотека JSON-server.

Next.js

Использовался Next.js в качестве фреймворка.

Zustand

Состояния приложения хранится в стейтменеджере Zustand.

Next-Auth (Google and Credentials)

Аутентификация пользователей реализована с помощью Next-Auth.

SSG and IRS

На проекте также использовались SSG и IRS.

Image Optimization

Для оптимизации изображений использовалась библиотека next/image.

Server Actions

Для обработки запросов к серверу также использовалась Server Actions.
